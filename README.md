# 🚀 LAN BeamDrop

**极速、极简的跨平台局域网文件与剪切板同步基站**

LAN BeamDrop 致力于打破操作系统之间的壁垒，在同一个局域网（Wi-Fi 或内网）下，无需安装任何客户端，即可实现 Windows、Mac、iOS、Android 和 HarmonyOS 之间的零延迟投送。

v1.0.0 正式版现已搭载 **单文件极客包 (Standalone Executable)**，从此告别 Node.js 环境配置，双击即用！

## ✨ 核心特性

- ⚡️ **开箱即用 (零配置)**：下载单文件版，双击直接启动。无需安装 Node.js、无需配置 npm，局域网内任意设备通过浏览器（或扫码）无缝接入。
- 📡 **mDNS 局域网雷达**：自带多播 DNS 广播，局域网内的设备无需记忆复杂的 IP 地址，雷达自动扫描，点开即连。
- 🖱️ **全局无缝拖拽**：支持文件拖拽上传与全局发光遮罩，一拖即传，并带有防误触机制。
- 📋 **剪切板闪现同步**：在一台设备上复制，另一台设备瞬间可见。彻底的 WebSocket 全双工通信，**不刷新、零延迟**。支持基于本地 JSON 的轻量级持久化存储，断电重启记录不丢失。
- 📁 **大文件分片引擎**：搭载智能前端分片与并发重试引擎，不再受限于内存大小，几个 G 的超大视频稳定传输，直接跑满本地路由器极限带宽。
- 🔍 **海量极速检索**：纯前端内存级的性能优化，原生 JS 驱动的超快缩略图引擎。支持按“图片/视频”、“文档”秒速分类检索。
- 📦 **优雅的批量收割**：勾选多张照片或文件，**一键打包成 `.zip` 瞬间提走**；用完即焚，一键批量销毁。
- 💬 **设备群聊体验**：基于 WebSocket 增量实时推送，设备连接瞬间打通，像用微信群聊一样优雅地双向通讯。
- 🔒 **绝对私密**：数据 100% 在本地局域网内流动，无外网服务器中转，安全可靠。
- ⚙️ **主机专属保护**：强大的本机专属高级设置面板，搭载严格的内网 IP 鉴权，支持随意更改落盘目录，甚至一键开启**防偷窥密码锁**。
- 🚀 **真机测速舱**：内置纯物理网卡极限吞吐测试组件与智能排障指南，一秒诊断 2.4G 拥堵与硬件瓶颈。

## 🛠️ 快速开始

### 方案 A：小白免安装版（强烈推荐 ✨）

这是最简单、最优雅的使用方式，适合绝大多数用户。

1. 前往 GitHub 的 [Releases 页面](../../releases/latest)。
2. 根据你的操作系统，下载对应的单文件原生程序：
   - Windows 用户下载：`lan-beamdrop-win.exe`
   - macOS 用户下载：`lan-beamdrop-macos`
   - Linux 用户下载：`lan-beamdrop-linux`
3. 放在任意目录，**双击运行**（或在终端中运行）。
4. 控制台会打印出一个巨大的二维码和局域网 IP 地址。手机或平板直接扫码即可瞬间连入投送室！

> 💡 **Tip**: 所有的业务数据和持久化剪切板都会安全地存放在你指定的落盘目录中，绝对不会污染你的系统盘。

### 方案 B：开发者极客版（需源码构建）

如果你想二次开发或体验最新的测试分支：

**环境要求**：[Node.js](https://nodejs.org/) (推荐 v18 及以上)

```bash
# 1. 克隆项目
git clone https://github.com/530china/lan-beamdrop.git
cd lan-beamdrop

# 2. 安装依赖
npm install

# 3. 启动本地热重载开发服务
npm run dev
```

## 📦 技术栈与架构

- **后端**: Node.js + Express + WebSocket (ws)
- **底层架构**: **esbuild** (模块合并去依赖) + **pkg** (V8引擎容器化封包) + GZip 强效压缩
- **前端**: 纯血原生 ES6 Modules / Vanilla JS / CSS (拒绝 React/Vue 等重型框架)
- **网络发现**: `bonjour-service` (mDNS 协议支持)
- **批处理与图形**: `archiver` (极速 ZIP 流) + `jimp` (原生图片缩略图引擎)
- **测试**: Jest + Supertest (TDD 驱动开发)

## 🚀 自动构建与发版 (CI/CD)
本项目已配置极度严苛的 GitHub Actions 工业级流水线。代码一旦推送到主干，将自动通过 `scripts/build.js` 编排 `esbuild` 与 `pkg` 交叉编译全平台二进制文件。

支持以下两种触发发版的方式：

1. **产品经理流 (Releases 面板)**: 在 GitHub 的 `Releases` 页面点击 "Draft a new release"，填入带有 `v` 的版本号 Tag（如 `v1.0.0`），点击 "Publish release"。后端流水线将自动拉起，打包完毕后自动将 `.exe` 挂载到该发布页的附件区。
2. **老板一键流 (Actions 面板)**: 进入 GitHub `Actions` 页面，选择 `CI/CD Pipeline`，点击 `Run workflow`，输入期望的版本号（如 `1.0.0`）并运行。流水线将全自动为你创建 Release 页面并打包分发。
