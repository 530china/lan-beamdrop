FIX: 修复批量打包下载中文双重编码与报错问题，新增 check-zip 预检与动态文件命名

WHY:
1. 前端生成的复选框 value 对中文进行了 encodeURIComponent 编码，导致提取文件名时发生双重编码，后端接收到乱码无法匹配真实文件。
2. 旧版代码调用了未定义的 toggleBatchMode 函数引发 ReferenceError，导致前端抛出虚假的“网络错误” Toast 提示。
3. 单击打包下载的 ZIP 压缩包名称固定，多次下载难以区分。
4. 直接下载纯文本消息无反馈，体验不佳。

WHAT:
1. 前端 (app.js): 在提取复选框文件路径时，加入 decodeURIComponent 正确还原中文文件名。
2. 前端 (app.js): 将错误的 toggleBatchMode 修正为 exitBatchMode()，消除网络错误伪报错。
3. 前后端 (app.js, files.js): 新增 /api/files/check-zip 接口，对选中的文件进行预先可用性检查。
4. 后端 (files.js): 通过 Content-Disposition 动态生成附带时间戳和类型的压缩包文件名。
5. 后端 (files.js): 完善对 req.query.files 数组格式的解析支持，规避文件名中含有特殊字符或逗号导致的截断 bug。

TEST:
1. 后端补充了 /check-zip 以及数组传参的 Jest 单元测试用例，所有 23 个核心测试全部绿灯通过。
2. 手动在浏览器使用 MCP 工具复现并确认了 DOM 值双重编码的根因，并在修复后通过浏览器 F5 刷新证实了中英文混合多文件打包下载完全顺畅可用。
3. 纯文本点击打包现在能正确弹出阻断提示。
