📱 手机连不上？网络排障指南

当您在电脑（PC）上运行本程序后，如果遇到 “手机或其他设备打不开网页”、“一直在加载中” 的情况，请按照以下步骤进行一分钟快速排查：

第一步：确认基础网络环境

  • 确保电脑和手机连接的是 同一个路由器 的 Wi-Fi 或网线。
  • 确保电脑 没有开启 VPN、代理软件（如 Clash、V2Ray 等可能会接管局域网流量）。
  • 核对手机的 IP 地址前三段是否与电脑的 IP 一致（例如电脑是 192.168.31.50，手机应该是 192.168.31.X）。

第二步：使用内建“网络诊断”工具

如果基础网络确认无误，但依然连不上。请在 电脑端 的主界面点击右上角 设置 ⚙️ -> 网络诊断 面板：
点击 🔍 扫描局域网设备 (ARP) 按钮，并在下方列表中寻找你手机的 IP 地址。

场景 A：列表中【找不到】手机的 IP 地址

结论：物理网络被阻断（极大概率是路由器的锅）
  • 原因 1：路由器开启了“AP 隔离”或“访客网络”。许多路由器为了安全，默认禁止 Wi-Fi 连接的设备互相通信。
  • 原因 2：双频隔离。你的电脑连的是 2.4G Wi-Fi，手机连的是 5G Wi-Fi，且路由器没有将两个频段桥接。
  • 原因 3：公共网络限制。如果是公司、学校、咖啡馆的 Wi-Fi，通常会强制在底层隔离所有设备。
👉 解决办法：
  1. 登录路由器后台，关闭“AP隔离 / 无线隔离”。
  2. 让电脑和手机连接同一个 Wi-Fi 频段（都连 5G）。
  3. 终极必杀技：直接用手机开热点，让电脑连接手机的热点，绝对能通！

场景 B：列表中【找到了】手机 IP，点击 Ping 测试【连通失败】

结论：网卡互相看见了，但数据包被丢弃（依然是路由器或网络的锅）

原因： 与场景 A 类似，部分路由器允许 ARP 广播寻找设备，但在实际传输数据（ICMP/TCP）时进行了拦截。

👉 解决办法： 同场景 A（关隔离 / 连同一个频段 / 手机开热点）。

场景 C：Ping 测试【✅ 连通成功】，但手机依然打不开网页！

结论：网络完美畅通，100% 是电脑 Windows 防火墙拦截了本程序！

原因： Windows 默认的防火墙策略极其严格。当本程序（EXE）尝试在局域网开放端口提供服务时，被 Windows Defender 默默拦截了外网（手机）发进来的请求。

👉 解决办法（防火墙放行）：
  1. 按下键盘 Win 键，搜索并打开 Windows Defender 防火墙。
  2. 点击左侧的 允许应用或功能通过 Windows Defender 防火墙。
  3. 点击右上角的 更改设置（需要管理员权限）。
  4. 在列表中向下滚动，找到本程序（名字可能是 lan-beamdrop.exe 或 Node.js）。
  5. 必须将其右侧的 “专用” 和 “公用” 两个复选框 全部打勾 ✅。
  6. 点击确定保存。手机刷新网页，瞬间秒开！

第三步：为什么局域网测速很慢？

如果您在主页点击 🚀 测速 后发现速度不理想，请对照以下原因排查：

嫌疑一：连接了 2.4G Wi-Fi（最常见）
原理： 2.4G 频段极其拥堵且理论上限低，实际测速通常只有 2MB/s ~ 8MB/s。而 5G 频段干扰少，可达 30MB/s ~ 100MB/s。

解决办法： 确保手机和电脑都连接在路由器名称带有 -5G 的 Wi-Fi 上。如果离路由器太远，建议直接 让手机开启个人热点，电脑连接手机热点，速度会瞬间起飞！
嫌疑二：Wi-Fi 半双工折损魔咒
原理： Wi-Fi 就像对讲机，同一时刻只能发或收。如果您的 电脑和手机都连着同一个 Wi-Fi，一份文件要先发给路由器，路由器再发给电脑，导致带宽直接腰斩一半。

解决办法： 最快、最稳的终极形态是：电脑插着网线，手机连接路由器的 5G Wi-Fi。这样两者都能跑满独立通道。
嫌疑三：电脑硬盘写入瓶颈（仅影响真实传大文件）
原理： 本工具自带的 🚀 测速功能 是纯网卡对传，不读写硬盘。但如果您测速有 80MB/s，真实传文件却只有 20MB/s，那就是电脑存储目录所在的硬盘太慢了。

解决办法： 老旧的机械硬盘（HDD）写入速度很慢，且容易卡顿。请在软件右上角的“设置”里，把 文件存储目录 更改到固态硬盘（SSD）所在的盘符。

💡 终极省流排障口诀：

  • 扫不到手机 ➡️ 路由器隔离了，手机开热点给电脑连！
  • 能扫到但 Ping 不通 ➡️ 路由器拦截了，手机开热点给电脑连！
  • 能 Ping 通但网页打不开 ➡️ 电脑防火墙拦截了，去控制面板放行这个 EXE！
