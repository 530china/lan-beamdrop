[ENH]: 新增局域网环境真机测速舱与排障指南

WHY:
为了解决用户在局域网下传输文件时由于 2.4G Wi-Fi 或网络环境拥堵导致“传输慢”的疑惑，需要一个不依赖硬盘读写、完全跨平台的纯物理带宽测试组件。同时需要一份深度的排障说明文档来安抚用户并引导其优化内网物理环境。

WHAT:
1. 【Backend】新增 `routes/speedtest.js` 并挂载至全局（无内网 IP 拦截）。下行接口利用 `Buffer.alloc` 推送无尽内存流；上行接口接收到分块后立即抛弃（黑洞模式）。完全不占用磁盘 I/O。
2. 【Frontend】顶部导航栏新增测速入口，弹出包含上下行双表盘（Gauge）的深色毛玻璃高级测速面板。
3. 【Frontend】利用 `ReadableStream` 读流与 `Uint8Array` 构建发包块，各进行 3 秒极限吞吐量测试，计算真实的网卡收发速率。
4. 【Frontend】根据速度极值给出绿黄红诊断结论，并在底层附加上了指向 `troubleshooting.html#speed` 的超链接。
5. 【Docs】在 `troubleshooting.html` 增补了对于 2.4G 惩罚、半双工衰减、以及 HDD 机械硬盘写入瓶颈的底层科普和实操极速方案（如手机开热点）。
TEST:
1. 在 Windows 10 (PC) 开启热点并桥接至手机 (Android) 的局域网环境下进行了完整吞吐压测。
2. 验证了下行测试（PC -> 手机）时流读取器 (ReadableStream) 每 0.1 秒精准拉取 Buffer 数据并实时反馈 UI 数值，无卡顿。
3. 验证了上行测试（手机 -> PC）高频并发推包的丢弃机制工作正常，未产生任何落盘文件，测速稳定在硬件极值。
4. 验证了双表盘 UI（毛玻璃特效）逻辑：下载与上传状态轮流激活，智能排障阈值判定（<10MB/s 标红，>30MB/s 标绿）触发精准。
5. 通过 `node -c public/app.js` 完成全部 JavaScript 代码环境的无错误（SyntaxError）静态验证。
